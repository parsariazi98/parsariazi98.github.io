Home / Polarization geolocalization

BioSensors Lab · Aug 2025–Present

Learning geolocation from polarization with physics built into the data pipeline.

This project starts from a simple practical constraint: real polarization data is expensive, but the underlying optics are structured. Instead of treating that structure as background knowledge, I used it to synthesize data and organize a two-stage prediction pipeline.

Goal Use polarization patterns to estimate location with stronger data efficiency.

Challenge

Turning physical structure into a training advantage

Polarization carries geometry tied to solar elevation, but collecting enough labeled data across time and location is slow. I designed a synthetic augmentation workflow that models those dependencies so the network sees richer variation before it ever reaches the final regression stage.

  • Modeled solar-elevation dependencies in polarization patterns to scale synthetic training data.
  • Used a Vision Transformer to predict solar-elevation trajectories.
  • Followed with a two-stage PointTransformer for geolocation regression.

Key metrics

  • ~750k polarization images in training.
  • < 0.5° latitude error on 35% of test days.
  • < 0.5° longitude error on 68% of test days.

Stack

ViT PointTransformer Synthetic augmentation

1. Build the right variation

The first move was not architecture, but data. The augmentation pipeline was designed to reflect how polarization structure changes with solar elevation instead of applying domain-agnostic perturbations.

2. Predict trajectory before location

A hierarchical design keeps the task organized: first infer the solar-elevation trajectory, then use a dedicated geolocation regressor to map that representation into latitude and longitude.

3. Evaluate on meaningful thresholds

Rather than only report average error, the project tracks how often the model stays under a strict sub-degree threshold across test days.
